## 요구사항 분석 

- 포인트 충전 / 조회 API
  - Point
    - 회원은 포인트를 충전하여 주문/결제를 한다.(계좌이체,카드x)
    - 주문 시 포인트를 차감한다.
    - 포인트 사용내역을 유저별로 관리한다.
> - 포인트 최대 금액은 100만원이다.
> - 포인트는 100원단위로 충전 가능하다.

- 상품 관리 API
  - Product
    - 상품정보를 등록한다. 
    - 상품정보를 수정한다.
    - 상품의 상태를 변경할수 있다.(판매가능,판매불가,삭제) 
    - 
- 재고 관리 API
  - 상품의 재고를 관리한다.
  - 상품 재고를 충전한다. 
  - 상품 재고를 차감한다.
  - 상품 입출고 내역을 관리한다. 어떻게..?
> - 재고는 음수가 될 수 없다.

```
- 재고의 경우 재고History가 따로필요한지 Order 테이블의 데이터로 파악하는지?
``` 
  
- 상품 조회 API
  - 상품 이름, 가격, 재고, 상태 업데이트 일시
  - 상품을 등록한다.
  - 상품정보를 변경한다.

```
예를들어 상품정보를 "삭제" 할때 DB에서 데이터를 지우는 하드딜리트와 상태값을 삭제상태로 변경하여 데이터를 노출하지 않게하는 소프트딜리트는 각각 어떤 상황에서 적용되나요?
```

- 주문 / 결제 API
  - Order
    - 주문 정보를 등록한다.
    - 주문 상태를 변경한다. 
    - 
```
  고객이 상품을 주문했을때 관리자가 알아야될 사항은 무엇일까? 
    고객 정보, 상품 정보, 상품 갯수, 쿠폰 적용 여부, 총가격, 주문일시, 주문 상태(결제,상품준비중,배송,배송완료,환불,교환)
    
 - 고객이 고객정보와 주문 정보를 넘긴다.
 - 주문정보(OrderItem): 상품정보, 상품갯수, 총가격, 주문일시, 상태
```

- 선착순 쿠폰 API
- 인기 판매 상품 조회 API
